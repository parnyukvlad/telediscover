Insert exactly this code (use `var` throughout — matches codebase style; this is a non-async synchronous function):

```js
function track(eventType, extra) {
  var initData = tg ? tg.initData : '';
  var payload = JSON.stringify(Object.assign({
    initData:       initData,
    event_type:     eventType,
    persona_handle: null,
    start_param:    null,
    utm_source:     null,
    utm_medium:     null,
    utm_campaign:   null,
  }, extra));
  var url = API_BASE + '/api/onlydate/track';
  var sent = false;
  if (navigator.sendBeacon) {
    sent = navigator.sendBeacon(url, new Blob([payload], { type: 'application/json' }));
  }
  if (!sent) {
    fetch(url, { method: 'POST', body: payload,
                 headers: { 'Content-Type': 'application/json' },
                 keepalive: true }).catch(function () {});
  }
}
```

Key implementation notes:
- `navigator.sendBeacon` requires the payload wrapped in `new Blob([payload], { type: 'application/json' })` — a plain string sends as `text/plain` which Hono's `c.req.json()` rejects with a parse error.
- `sent = navigator.sendBeacon(...)` — the `if (!sent)` fallback triggers when sendBeacon returns `false` (payload too large, page unloaded) OR when sendBeacon is unavailable in older Telegram WebViews. The `if (!sent)` checks the return value, not just availability.
- `keepalive: true` on the fetch fallback allows it to survive navigation events.
- `.catch(function () {})` — all failures are silently swallowed. The user experience must never degrade if tracking fails.
- `tg` is referenced by name — `track()` is defined before `const tg = ...` in source order, but it is only CALLED after `tg` is initialized (inside `init()` which runs last). No hoisting issue because `tg` is a `const` and `track()` is a function declaration (hoisted).

Wait — `tg` is `const` not `var`, so it is NOT accessible in `track()` at function-definition time. Since `track()` is called only from within `init()` and event handlers (all after `tg` is set), this works correctly. The `var initData = tg ? tg.initData : '';` line safely handles the case where `tg` is null (outside Telegram).

```js
function init() {
  /* ── Attribution: read once, persist across WebView hot-reloads ── */
  var attr = {};
  var stored = sessionStorage.getItem('od_attr');
  if (stored) {
    try { attr = JSON.parse(stored); } catch (e) { attr = {}; }
  } else {
    var sp = new URLSearchParams(window.location.search);
    var tgStartParam = tg && tg.initDataUnsafe && tg.initDataUnsafe.start_param;
    attr = {
      start_param:  tgStartParam                  || null,
      utm_source:   sp.get('utm_source')           || null,
      utm_medium:   sp.get('utm_medium')           || null,
      utm_campaign: sp.get('utm_campaign')         || null,
    };
    sessionStorage.setItem('od_attr', JSON.stringify(attr));
  }
  track('session_start', attr);

  /* ── Safe-area JS override (for old Telegram that returns 0 from CSS env()) ── */
  if (tg && tg.contentSafeAreaInset && tg.contentSafeAreaInset.top > 0) {
    $tabBar.style.paddingTop = tg.contentSafeAreaInset.top + 'px';
  }

  renderSkeletons();
  attachTabEvents();
  ...rest of existing init() body unchanged...
```

Key notes for the attribution block:
- `sessionStorage.getItem('od_attr')` is checked FIRST — if data exists from a previous hot-reload, use it. `tg.initDataUnsafe.start_param` is lost after WebView hot-reload (Pitfall 4).
- `try/catch` wraps `JSON.parse(stored)` in case of corrupt sessionStorage data — falls back to `{}` (empty object).
- `tg && tg.initDataUnsafe && tg.initDataUnsafe.start_param` — triple guard because `initDataUnsafe` may be null outside Telegram.
- `track('session_start', attr)` — fires immediately with all four attribution fields. `Object.assign` inside `track()` merges attr into the base payload, so null values for missing fields are sent (backend expects all fields, nulls accepted).
- The safe-area JS override uses `$tabBar` (already declared as a const at line ~568). Reads `contentSafeAreaInset.top` after `tg.ready()` has already been called (it was called before `init()` at the top of the script block).
- Use `var` for `attr`, `stored`, `sp`, `tgStartParam` — matches codebase non-async function style.
