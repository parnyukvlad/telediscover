**Edit 1 — Viewport meta (line 5):**
Change:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />
```
To:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover" />
```
This populates `env(safe-area-inset-*)` CSS variables on iOS (required for D-04 and D-05 to work). Without this, safe-area-inset-top/bottom always resolve to 0.

**Edit 2 — #grid-view and #profile-view min-height (lines ~40-41):**
Change:
```css
#grid-view    { display: block;  min-height: 100vh; }
#profile-view { display: none;   min-height: 100vh; background: var(--bg); }
```
To:
```css
#grid-view    { display: block;  min-height: 100vh; min-height: 100dvh; }
#profile-view { display: none;   min-height: 100vh; min-height: 100dvh; background: var(--bg); }
```
The stacked declaration pattern (100vh then 100dvh) ensures older iOS falls back to 100vh while modern devices get 100dvh (correct behavior when browser chrome changes height dynamically). Per D-02.

**Edit 3 — Add #app-wrapper CSS rule** (add immediately after the existing `#grid-view` / `#profile-view` lines in the `/* ── Views ── */` section):
```css
#app-wrapper {
  max-width: min(100vw, calc(100dvh * 9 / 16));
  margin: 0 auto;
  position: relative;
  min-height: 100vh;
  min-height: 100dvh;
  background: var(--bg);
}
```
Per D-03. `position: relative` alone does NOT create a containing block for fixed positioning (no transform/filter/perspective on this element — the lightbox is safe). The `background: var(--bg)` fills the column so it matches the dark body background seamlessly.

**Edit 4 — Safe-area padding on .tab-bar and .profile-topbar:**

For `.tab-bar`, change `padding: 8px 12px 10px;` to:
```css
padding: 8px 12px 10px;
padding-top:    max(8px,  env(safe-area-inset-top));
padding-bottom: max(10px, env(safe-area-inset-bottom));
```
(Three separate padding lines: the base `padding` shorthand sets all four sides, then the two subsequent `padding-top` and `padding-bottom` override vertical paddings with safe-area-aware values using `max()` so the minimum padding is preserved when safe-area-inset is 0. Per D-04 and D-05.)

For `.profile-topbar`, change `padding: 16px 16px 10px;` to:
```css
padding: 16px 16px 10px;
padding-top: max(16px, env(safe-area-inset-top));
```
(The base shorthand stays; the override applies only when the topbar is visible, e.g. on notched devices.)
