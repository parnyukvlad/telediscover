**Step 1: Rewrite `apps/onlydate-worker/src/index.ts`**

Replace the entire file with this thin assembly. The CORS middleware block (lines 20-27 of original) is preserved verbatim. All route logic is removed (it now lives in routes/).

```typescript
import { Hono } from 'hono';
import adminRoutes   from './routes/admin';
import publicRoutes  from './routes/public';
import webhookRoutes from './routes/webhook';

interface Env {
  DB: D1Database;
  BOT_TOKEN: string;
  MEDIA: R2Bucket;
  ADMIN_PASSWORD: string;
}

const app = new Hono<{ Bindings: Env }>();

// ─── CORS ─────────────────────────────────────────────────────────────────────
app.use('*', async (c, next) => {
  c.header('Access-Control-Allow-Origin', '*');
  c.header('Access-Control-Allow-Methods', 'GET, POST, OPTIONS');
  c.header('Access-Control-Allow-Headers', 'Content-Type, X-Admin-Password');
  if (c.req.method === 'OPTIONS') return c.body(null, 204);
  await next();
});

// ─── Routes ───────────────────────────────────────────────────────────────────
app.route('/', publicRoutes);
app.route('/', adminRoutes);
app.route('/', webhookRoutes);

// ─── 404 ─────────────────────────────────────────────────────────────────────
app.notFound((c) => c.json({ error: 'Not found' }, 404));

export default app;
```

- The new `Env` interface contains `ADMIN_PASSWORD: string` (per D-12)
- There is NO `const ADMIN_PASSWORD` string literal anywhere
- The `export default app` line is present (required by Cloudflare Workers)
- The CORS middleware content is character-for-character identical to the original (same headers, same method list, same OPTIONS response)

**Step 2: Update `apps/onlydate-worker/wrangler.toml`**

Add the `[secrets]` section declaring `ADMIN_PASSWORD`. Read the current wrangler.toml first, then append:

```toml
# Secrets (values set via: wrangler secret put ADMIN_PASSWORD)
[secrets]
ADMIN_PASSWORD = ""
```

Wait — D1 wrangler.toml uses `[[d1_databases]]` not `[secrets]`. The correct way to declare a secret binding in Wrangler v3 is to add it to `[vars]` for documentation purposes or simply note it in a comment. Actually, secrets are NOT declared in wrangler.toml — they are set via `wrangler secret put` and automatically available on `c.env`. The correct step is:

Add a comment to wrangler.toml documenting the required secret:

```toml
# Secrets (set via CLI — not stored in this file):
#   wrangler secret put ADMIN_PASSWORD
```

Add this comment after the `[vars]` block. The secret will be available as `c.env.ADMIN_PASSWORD` at runtime once set via `wrangler secret put ADMIN_PASSWORD`.

The final wrangler.toml should look like:

```toml
name = "onlydate-api"
main = "src/index.ts"
compatibility_date = "2024-05-01"
compatibility_flags = ["nodejs_compat"]

[vars]
ENVIRONMENT = "production"

# Secrets (set via CLI — not stored in this file):
#   wrangler secret put ADMIN_PASSWORD

# Prod D1 — read-only access for public discovery
```
