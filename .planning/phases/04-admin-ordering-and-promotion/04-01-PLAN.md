Add three changes to admin.ts:

1. **New endpoint: POST /api/onlydate/admin/feed-entries/reorder**
   - Route path: `/api/onlydate/admin/feed-entries/reorder`
   - Auth: `if (!isAdmin(c)) return c.json({ error: 'Unauthorized' }, 401);`
   - Body: `{ order: string[] }` — array of feed_entry_id values in desired display order
   - Validate: `Array.isArray(body.order) && body.order.length > 0`, else 400
   - Implementation: Build array of prepared statements, one per ID:
     ```
     const stmts = body.order.map((id, i) =>
       c.env.DB.prepare('UPDATE onlydate_feed_entries SET sort_order = ? WHERE id = ?').bind(i + 1, id)
     );
     await c.env.DB.batch(stmts);
     ```
   - Use `c.env.DB.batch(stmts)` for atomic update (NOT individual awaits)
   - Success: `return c.json({ ok: true })`
   - Error: `console.error('[OnlyDate] reorder error:', err);` then 500
   - Place this endpoint BEFORE the existing `GET /api/onlydate/admin/personas` route

2. **New endpoint: POST /api/onlydate/admin/feed-entry/toggle-promoted**
   - Route path: `/api/onlydate/admin/feed-entry/toggle-promoted`
   - Auth: same isAdmin check
   - Body: `{ feed_entry_id: string, is_promoted: boolean }`
   - Validate: `feed_entry_id` required, else 400
   - Implementation:
     ```
     const val = body.is_promoted ? 1 : 0;
     await c.env.DB.prepare('UPDATE onlydate_feed_entries SET is_promoted = ? WHERE id = ?')
       .bind(val, body.feed_entry_id).run();
     ```
   - Success: `return c.json({ ok: true })`
   - Error: `console.error('[OnlyDate] toggle-promoted error:', err);` then 500
   - Place this endpoint right after the reorder endpoint

3. **Update admin personas query (GET /api/onlydate/admin/personas)**
   - In the feed_entry branch of the UNION ALL query, add two new columns:
     - `fe.sort_order AS sort_order`
     - `fe.is_promoted AS is_promoted`
   - In the personas branch, add matching synthetic columns:
     - `NULL AS sort_order`
     - `0 AS is_promoted`
   - Change `ORDER BY` from `persona_name ASC` to `sort_order ASC, persona_name ASC` (so feed entries appear in admin-set order, personas alphabetically after)
   - In the `PersonaEntry` type, add `sort_order` and `is_promoted` fields:
     ```
     type PersonaEntry = { id: string; name: string; username: string; is_active: boolean; feed_visible: number | null; source: string; photos: PhotoRow[]; cover_url?: string; sort_order: number | null; is_promoted: number };
     ```
   - In the map-building loop, extract and assign:
     ```
     sort_order: row.sort_order as number | null,
     is_promoted: (row.is_promoted as number) ?? 0,
     ```

Follow existing code patterns: try/catch wrapping, `[OnlyDate]` log prefix, same body parsing pattern as other endpoints.
